Introduction to version control

The need for version control

When you’re developing code, you need version control to address the following issues:

  • How do I make incremental changes and share my work with others?
  • How do I go back to the version of this file from?
  • What changed between version X and version Y of a file?
  • People have been making changes to the same file that I’m working on. How do I reconcile and merge all these changes?

What is version control?

A version control system manages changes to a set of files and keeps a history of changes. It enables you to experiment with your code safely, and to undo mistakes.

Version control is similar to: - Snapshots of VMs - Incremental backups of files - Wiki versioning
